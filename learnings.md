## 2026-04-04

**Skills check must happen even when the answer is "none apply."**
On the session that introduced the skills-check rule, I didn't actually consult skills-index.md before starting work. The correct habit is: open the index, scan it, decide consciously that nothing applies (or load what does). Skipping the check because the task "seems simple" defeats the purpose.

**Skills check added to workflow.**
Before this session there was no habit of loading relevant skills before starting work. The new rule in CLAUDE.md and skills-index.md address this gap. Going forward, consult skills-index.md at the start of every task and invoke relevant skills before proceeding.

**Session learning habit formalised.**
There was no end-of-session review habit. learnings.md and the new CLAUDE.md rule address this. If a session surfaces a clear capability gap, recommend a skill to address it in the handoff note.
